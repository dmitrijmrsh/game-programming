# Deep Canyon — Исследование затонувшего каньона

3D-игра на Unity 6, в которой ты управляешь глубоководным батискафом, спускаешься в затонувший каньон и собираешь светящиеся Эхо-Кристаллы

## Ссылка на видеозапись игрового процесса

Google диск: https://drive.google.com/file/d/1ONDTyL32001wJu-aht8krG1keMb0ShKI/view?usp=drive_link

## Требования

- **Unity 6** (версия `6000.4.0f1` или совместимая)
- **Universal Render Pipeline (URP)** — устанавливается автоматически через пакеты

## Как запустить проект

### 1. Установить Unity

1. Скачай [Unity Hub](https://unity.com/download)
2. В Unity Hub нажми **Installs → Install Editor**
3. Выбери версию **Unity 6 (6000.4.x)**. Если точной версии нет — бери ближайшую `6000.4.x`
4. При установке убедись, что отмечены модули:
  - **Windows Build Support** (если собираешь под Windows)
  - **Mac Build Support** (если собираешь под macOS)

### 2. Подготовить и открыть проект

1. Скопируй всю папку проекта на свой ПК (или распакуй архив / клонируй репозиторий)
2. **Удали следующие папки** из корня проекта, если они есть (они генерируются автоматически):
  - `Library/`
  - `Temp/`
  - `Logs/`
  - `obj/`
3. Открой **Unity Hub → Projects → Open → Add project from disk**
4. Выбери корневую папку проекта (ту, где лежат `Assets/`, `Packages/`, `ProjectSettings/`)
5. Unity предложит открыть проект — нажми **Open**
6. Дождись импорта (первый раз может занять 3–10 минут, Unity пересоздаст `Library/` заново)

### 3. Собрать сцену

Сцена генерируется программно через скрипт:

1. В верхнем меню Unity нажми **DeepCanyon → Build Entire Scene**
2. Дождись сообщения `[DeepCanyon] Scene built and saved!` в консоли
3. Сцена сохранится в `Assets/DeepCanyon/Scenes/DeepCanyon.unity`

### 4. Запустить игру

Нажми кнопку **Play** (▶) в верхней панели Unity Editor.

## Управление


| Клавиша   | Действие                                          |
| --------- | ------------------------------------------------- |
| **WASD**  | Движение батискафа (вперёд/назад/влево/вправо)    |
| **Q / E** | Подъём / погружение                               |
| **Мышь**  | Поворот камеры                                    |
| **F**     | Включить/выключить луч (для приманивания существ) |


## Три основные механики

### Динамическое давление и плавучесть

Чем глубже погружаешься, тем сильнее давление. Управление усложняется, корпус получает урон, появляются визуальные протечки. Следи за полоской HP корпуса вверху экрана.

### Подводные течения

Цветные зоны течений видны по потокам частиц и цветной подсветке:

- **Зелёные зоны** — помогают (Assist), ускоряют движение с баффом Speeder
- **Красные/оранжевые зоны** — опасные (Hazard), сильно сносят без баффа Armorer

### Симбиоз с существами

В каньоне плавают светящиеся существа четырёх типов. Наведи на них луч (**F**), чтобы они начали следовать за тобой. Каждый тип даёт свой бафф:

- 🟢 **Healer** (зелёный) — восстанавливает HP корпуса
- 🟡 **Speeder** (жёлтый) — ускорение + усиление Assist-течений
- 🔵 **Armorer** (синий) — снижает урон от давления + ослабляет Hazard-течения
- 🟣 **Scout** (фиолетовый) — подсвечивает ближайшие кристаллы

## Цель игры

Собрать все **13 Эхо-Кристаллов**, разбросанных по каньону от поверхности до самого дна (−250м). Счётчик кристаллов — в правом верхнем углу.

## Сборка standalone-билда

Если хочешь собрать .exe / .app для запуска без Unity Editor:

1. **File → Build Settings**
2. Убедись, что сцена `Assets/DeepCanyon/Scenes/DeepCanyon.unity` добавлена в список (если нет — нажми **Add Open Scenes**)
3. Выбери целевую платформу (Windows / macOS / Linux)
4. Нажми **Build** или **Build And Run**
5. Выбери папку для сохранения билда

## Структура проекта

```
Assets/DeepCanyon/
├── Input/              — Input Actions (управление)
├── Scenes/             — Сгенерированная сцена
├── Scripts/
│   ├── Bathyscaphe/    — Контроллер, давление, плавучесть, корпус
│   ├── Creatures/      — Flocking AI, луч, баффы, VFX симбиоза
│   ├── Currents/       — Векторное поле течений, зоны, VFX binder
│   ├── Editor/         — Скрипт генерации сцены
│   └── Game/           — GameManager, HUD, пост-обработка
├── Tests/EditMode/     — Юнит-тесты
├── Settings/           — Volume Profile (пост-обработка)
└── VFX/Currents/       — VFX Graph для течений
```
